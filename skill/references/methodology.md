# 浏览方法论

## 浏览哲学

**像人一样思考，兼顾高效与适应性。**

**① 定义成功标准** — 用户要做什么？什么算完成？

**② 选择起点** — 根据任务性质选最可能直达的方式。需要登录态/已知反爬平台 → 直接 CDP。

**③ 过程校验** — 每一步的结果都是证据。发现方向错了立即调整，不在同一方式上反复重试。搜索没命中可能是"目标不存在"，不是"还没找对方法"。

**④ 完成判断** — 对照成功标准，完成即停，不过度操作。

## 程序化操作与 GUI 交互

- **程序化**（URL 导航、eval 操作 DOM）：快、精确，但容易触发反爬
- **GUI 交互**（点击、填写、滚动）：确定性最高，不会被限制，但慢

当程序化方式受阻时，GUI 交互是可靠的兜底。

**站点内 URL 的可靠性**：DOM 中的 href 携带完整上下文，手动构造的 URL 可能缺失隐式参数导致被拦截。

## 信息核实

核实目标是**一手来源**，不是更多二手报道。多个媒体引用同一错误会造成循环印证假象。

| 信息类型 | 一手来源 |
|----------|---------|
| 政策/法规 | 发布机构官网 |
| 企业公告 | 公司官方新闻页 |
| 学术声明 | 原始论文/机构官网 |
| 工具能力/用法 | 官方文档、源码 |

找不到官网时，用权威媒体原创报道（非转载）作为次级依据，但需向用户说明来源局限性。

## 并行调研：子 Agent 分治

多个独立调研目标时，分治给子 Agent 并行执行。

**好处**：速度（并行）、上下文保护（主 Agent 只接收摘要）

**并行 CDP**：所有子 Agent 共享一个 Chrome + Proxy，通过不同 targetId 操作不同 tab，无竞态。

**子 Agent Prompt 原则**：
- 写 `必须加载 web-access skill 并遵循指引`
- 描述**要什么**，不限定**怎么做**
- 避免暗示具体手段的动词（「搜索xx」→「获取xx的信息」

| 适合分治 | 不适合分治 |
|----------|-----------|
| 目标独立、互不依赖 | 有依赖关系 |
| 每个子任务量大 | 简单单页查询 |
| 需要 CDP 或长时间运行 | baidu-search 能搞定的 |
